# Elixir Web弾幕シューティング計画書

## プロジェクト名

- 正式名: `danmaku`
- Phoenix アプリ名も `danmaku` を前提にする

## 目的

Docker を前提とした開発環境で Elixir を使ったWebゲームを開発し、最終的に2Dゲーム性を持ちながら3D表示で見せる弾幕シューティングとして完成させる。

## 開発方針

- ゲームロジックはできる限り `Elixir` 側で管理する
- 表示はブラウザ側で行う
- ゲーム性は2D、見た目は3Dの `2.5D` 構成にする
- 最初は最小構成で作り、後から弾幕密度や演出を強化する

詳細な技術構成は [ARCHITECTURE.md](ARCHITECTURE.md) を参照する。

## ゲームデザイン方針

- 東方Project風の縦スクロール弾幕シューティングを目指す
- 自機は小さく、見た目より小さい当たり判定を持たせる
- 弾幕は理不尽さよりも規則性と美しさを重視する
- ボス戦とフェーズ進行を重視する
- 3D演出は視認性を壊さない範囲で使う

## 開発フェーズ

### 1. 環境構築

- Docker 開発環境を用意する
- `Elixir + Phoenix + Node` を含む `app` コンテナを作る
- `Phoenix Channels` が使える構成にする
- 3D描画ライブラリを導入できる状態にする

### 2. 最小ゲーム試作

- 1画面の2Dゲーム空間を3D表示する
- 自機を1つ表示する
- キー入力をElixirへ送信する
- Elixir側で座標更新する
- 画面外へ出ないようにする

### 3. シューティング化

- 自機ショットを実装する
- 敵を1種類出現させる
- 敵HP、撃破、スコアを実装する

### 4. 弾幕の土台

- 敵弾を実装する
- 当たり判定を実装する
- 被弾、残機、ゲームオーバーを実装する
- 被弾後の無敵時間を実装する
- 状態同期の負荷を測定する

### 5. 弾幕らしさの強化

- 扇状、円形、渦巻き、時間差発射などの基本パターンを追加する
- 敵の移動パターンを増やす
- ボスと複数フェーズを実装する

### 6. Webゲームとして仕上げる

- タイトル画面を作る
- リトライ機能を作る
- セッション中のスコア表示を整える
- 音を追加する
- PC前提UIとして調整する

## 最初の2週間の目標例

- 1日目: Docker 開発環境を用意する
- 2日目: `Phoenix` アプリ作成と `Channels` 接続確認を行う
- 3-4日目: 3Dシーン上に自機表示と状態受信を実装する
- 5-6日目: 入力送信とElixir側移動を実装する
- 7-8日目: 自機ショットを実装する
- 9-10日目: 敵1体、当たり判定、状態同期を実装する
- 11-14日目: 敵弾、ゲームオーバー、性能確認を実装する

## 最初に決めること

- まずはシングルプレイ専用にするか
- 見下ろし固定の縦スクロールSTGにするか
- サーバ更新周期を何fps相当にするか

## 現時点の推奨判断

まずは Docker を開発環境の正本として採用し、`Phoenix` を土台にして、Elixir主導でゲーム状態を管理し、フロントエンドは `Babylon.js` で3D表示を担当する構成を採用する。
最初は小さく成立させ、あとから弾幕、演出、ボス戦を段階的に追加する。
