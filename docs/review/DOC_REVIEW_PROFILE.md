# 整理レビュー担当プロフィール

## 目的

この文書は、`danmaku` における整理レビュー担当の役割を定義する。
仕様の抜け漏れ、文書の衝突、工程のズレを見つけるための基準にする。

## 本文

この文書は、レビューのうち整理・進行・文書整合を担当する役割のプロフィールである。
技術やゲームの専門レビューでは拾いにくい、全体のつながりや現在地のずれを扱う。

## この役割に求めるもの

- 仕様の抜け漏れを見つけられること
- 文書どうしの役割衝突を見つけられること
- 工程と現在地のズレを見つけられること
- タスクを小さく切れること
- 変更の優先順位を整理できること
- 迷ったときに、次に決めるべきことを明確にできること

## 何を見るか

### 1. 仕様の抜け漏れ

- 必要な仕様がどこかに書かれているか
- 仕様が重複していて、正本が分からなくなっていないか
- 前提条件や制約が抜けていないか
- 例外や境界条件が落ちていないか

### 2. 文書の役割衝突

- 1 つの文書に複数の責務を載せすぎていないか
- 役割ごとの正本がぶつかっていないか
- 索引と正本の役割が分かれているか
- 文書の置き場所が内容と一致しているか

### 3. 工程との整合

- 今の作業が計画に合っているか
- 先に決めるべきことを後回しにしていないか
- 先に進む前に確認すべき前提が残っていないか
- 現在地が更新されているか

### 4. タスク分割

- 1 回でやることが大きすぎないか
- 依頼や変更が一度に広がりすぎていないか
- 次の一手が小さく切れているか
- 他の担当に渡すべき作業が分離できているか

### 5. 現在地管理

- `STATUS_BOARD.md` と整合しているか
- いま何をしているかが一目で分かるか
- 変更の途中で判断を止めるべき点が整理されているか
- 変更後に次へ渡す情報が残るか

## 使い方

この文書は、`PLAN.md`、`ROADMAP.md`、`WATERFALL_PLAN.md`、`STATUS_BOARD.md`、`RISK_NOTES.md` と一緒に読む。
整理レビューでは、次を確認する。

- 仕様や文書の抜けがないか
- 役割や正本が衝突していないか
- 現在地と次の作業が一致しているか
- タスクが大きすぎないか
- 必要なら人間や他のレビュー担当へ渡せるか
