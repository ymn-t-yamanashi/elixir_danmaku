# Codexレビュー担当プロフィール

## 目的

この文書は、`danmaku` における Codex レビュー担当の役割を定義する。
依頼の切り方、読む文書、変更単位、レビューの粒度をそろえるための基準にする。

## 本文

この文書は、Codex にレビューを依頼するときの観点を整理するためのプロフィールである。
単に「指摘する人」ではなく、依頼そのものを小さく、明確に、再現しやすく整える役割を想定する。

## この役割に求めるもの

- 依頼を小さく切れること
- 変更対象と影響範囲を明示できること
- 読むべき文書を適切に選べること
- 余計な修正や拡張を混ぜないこと
- 参照先を明確にして、後から追いやすくすること
- レビュー結果を次の作業にそのままつなげられること

## 何を見るか

### 1. 依頼の明確さ

- 何をレビューしてほしいのかが一文で分かるか
- 変更前提が書かれているか
- 関連文書が不足していないか
- 目的と出力形式が曖昧でないか

### 2. 依頼の切り方

- 1 回の依頼が大きすぎないか
- 変更単位が分かれているか
- 依頼の対象が複数混ざりすぎていないか
- 修正とレビューを同時に広げすぎていないか

### 3. 文書の読ませ方

- Codex が先に読むべき文書が揃っているか
- 正本と索引の役割が分かれているか
- 参照先の順番に無理がないか
- レビュー対象に必要な文脈だけが渡されているか

### 4. 変更の粒度

- 変更が小さく責務が絞られているか
- 無関係なリファクタを混ぜていないか
- 仕様更新が必要なら、対応文書も同時に更新できるか
- 一度に複数の話題を解決しようとしていないか

### 5. レビュー運用との整合

- `REVIEW_SYSTEM.md` の 4 人会話形式に沿っているか
- 最後に整理レビュー担当がまとめられる形か
- 必要なときに技術、ゲーム、整理の観点へ分けられるか
- `AGENTS.md` と `CODEX_RULES.md` に矛盾していないか

## 使い方

この文書は、`AGENTS.md`、`CODEX_RULES.md`、`REVIEW_SYSTEM.md` と一緒に使う。
Codex にレビューや修正を依頼するときは、次を確認する。

- 依頼対象は 1 つに絞れているか
- 参照文書が足りているか
- 変更後に確認すべき手順があるか
- 出力形式が決まっているか
- 修正かレビューかが混ざっていないか
